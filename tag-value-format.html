3 Tag-Value Format

This section describes a simple, text based, format for representing the instances of the SPDX abstract model.

The SPDX tag-value format is a line based format. Each line starts with a "tag". The tag portion of the line is terminated by a ":". The value of the tag is remainder of the line after the ":". Multi-line values are supported by the use of a triple quote mechanism. If the sequences """ is first non-whitespace characters of a value, the value is consider to be the remainder of that line and any further lines until a the """ sequence is encountered again.

Example 1

SPDXVersion: SPDX-1.0
CreatedBy: Tool: spdx-gen 1.0
Example 2

SPDXVersion: SPDX-1.0
Description: """This
is along
multiline value"""

3.1 Grammar

The syntax of SPDX files described formally is:


      tagvaluefile   = *tagandvalue
      tagandvalue    = tag *WSP ":" *WSP value
      tag            = *tagchar
      tagchar        = ALPHA / DIGIT / "-" / "_" / "."
      value          = multilinevalue / *UNICODEVCHAR *WSP CRLF
      multilinevalue = 3DQUOTE *UNICODEVCHAR 3DQUOTE *WSP CRLF

3.2 Mapping to RDF

Tags map to RDF properies. For example, the line CreatedBy: Tool: spdx-gen 1.0 represents an RDF propery whose subject is the SDPX document of the file being parsed, with the predicate http://spdx.org/vocab#CreatedBy and the value "Tool: spdx-gen 1.0".
