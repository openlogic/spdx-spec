3. Common Overview Information

One instance of the Common Overview Information is required per package being analyzed. A package can contain sub-packages, but the overview is a reference to the entire contents of the package listed.

Fields:

    1. Formal Name

      1. Purpose: Full name of the package as given by originator with version information if available.

      2. Intent: Here, the formal name of each package is an important conventional technical identifier to be maintained for each package.

      3. Cardinality: Mandatory single instance.

      4. Tag: "DeclaredName:"

      5. RDF: /RDF/SPDXDoc/Describes/Package/DeclaredName

      6. DataFormat: text string of full name and version information if available.

      7. Example: DeclaredName: glibc 2.11.1

    2. Specific Package File Name

      1. Purpose: File name of package instance.

      2. Intent: Here, the actual filename of the compressed file (containing the package) is a significant technical element that needs to be carried with each package's Identification Information.

      3. Cardinality: Mandatory single instance

      4. Tag: MachineName:

      5. RDF: /RDF/SPDXDoc/Describes/Package/MachineName

      6. Data Format: text string of machine generated file name and version typically includes the packaging and compression methods used.

      7. Example: MachineName: glibc-2.11.1.tar.gz

    3. Unique Identifier

      1. Purpose: This field provides an independently reproducible mechanism that permits unique identification of a specific package that correlates to the data in this SPDX file. This identifier enables a recipient to determine if any file in the original package has been changed. The SHA1 unique identifier mechanism will be used.

      2. Intent: Here, by providing an unique identifier of each package, confusion over which version/modification of a specific package the Identification Information references should be eliminated.

      3. Cardinality: Mandatory single instance

      4. Tag: "SHA1"

      5. RDF: /RDF/SPDXDoc/Describes/Package/SHA1

      6. Data Format: 160 bit value represented as 40 hexadecimal digits.

      7. Example: SHA1: d6a770ba38583ed4bb4525bd96e50461655d2758

    4. Download URL

      1. Purpose: This field identifies the download Universal Resource Locator (URL) for this package at the time that the SPDX file was initially created and the analysis was done. If there is no public URL, then it is explicitly marked as “unknown”.

      2. Intent: Here, where to download the exact package being referenced is a critical verification and tracking datum.

      3. Cardinality: Mandatory single instance

      4. Tag: URL:

      5. RDF: /RDF/SPDXDoc/Describes/Package/URL/@rdf:resource

      6. Data Format: uniform resource locator (when the URL is explicitly “unknown” use http://spdx.org/rdf/unknown)

      7. Example: URL: http://ftp.gnu.org/gnu/glibc/

    5. Additional Source Information

      1. Purpose: This is a free form text field that contains additional comments about the origin of the package. For instance, this field might include comments indicating whether the package been pulled from a source code management system or has been repackaged.

      2. Intent: Here, by providing a freeform field, reviewers can provide any additional information to describe any anomalies, or discoveries, in the determination of the origin of the package.

      3. Cardinality: Optional single instance

      4. Tag: "SourceInfo:"

      5. RDF: /RDF/SPDXDoc/Describes/Package/SourceInfo

      6. Data Format: free form text without line separator

      7. Example:

    6. Declared License(s) for a Package

      1. Purpose: This field lists the licenses that have been declared by the creators of the package. The licenses are listed using a standard short form name. See Appendix I for standardized license short forms. If a Declared License is not one of the standardized license short forms, this field must contain a reference to the full license text included in this SPDX file. If no license is declared, the “NotSpecified” short form in Appendix I should be used. If it is unclear if there is a license reference or not, use “Unknown” short form from Appendix I. If more than one license is formally declared for package, then each should be listed. If the creators of the package offer the recipent a choice of licenses, then each of the choices will be declared as a “disjunctive” license.

      2. Intent: This is simply the license identified in text in the actual package source code files (typically in the header of each package file.) This field may have multiple declared licenses, if multiple licenses are declared at the package level.

      3. Cardinality: Mandatory, one or multiple.

      4. Tag: "DeclaredLicense:"

      5. RDF:

        /RDF/SPDXDoc/Describes/Package/DeclaredLicense

        /RDF/SPDXDoc/Describes/Package/DeclaredLicense/DisjunctiveLicense

      6. Data Format: <short form identifier in Appendix I> | "FullLicense"-N

      7. Example: DeclaredLicense: LGPL-2.0

    7. Detected License(s) in Package

      1. Purpose: This field list of all licenses found in the package by scanning the files, manually or using automated tools. The licenses should use the standard short form names. See Appendix I for standardized license short forms. If a Detected License is not one of the standardized license short forms, this field must contain a reference to the full licenses text included in this SPDX file. If no licenses are detected then the “NotSpecified” short form in Appendix I should be used. If it is unclear if there is a license reference or not, use “Unknown” short form from Appendix I.

      2. Intent: Here, we intend to capture all licenses within the package detected by any scanning tools used by the package reviewer.

      3. Cardinality: Mandatory, one or multiple.

      4. Tag: "DetectedLicense:"

      5. RDF: /RDF/SPDXDoc/Describes/Package/DetectedLicense/

      6. Data Format: <short form identifier in Appendix I> | "FullLicense"-N

      7. Example:

        DetectedLicense: GPL-2.0

        DetectedLicense: FullLicense-1

        DetectedLicense: FullLicense-2


    8. Declared Copyright Holder of Package

      1. Purpose: Identify the author(s) and licensor(s) of package, as well as any dates present. This will be a free form text field extracted from the package information files.

      2. Intent: Here, by identifying the actual author(s), some ambiguities, e.g., under which license the author(s) were intending to license the package, may be resolvable by knowing who to contact for clarity.

      3. Cardinality: Mandatory, single instance

      4. Tag: "DeclaredCopyright:"

      5. RDF: /RDF/SPDXDoc/Describes/Package/DeclaredCopyright

      6. Data Format: free form text | "unknown"

      7. Example: DeclaredCopyright: Copyright 2008-2010 John Smith

    9. Short Description

      1. Purpose: This field is a short description of the package

      2. Intent: Here, the intent is to allow a reader/reviewer of this field to quickly understand the function/use of the package, at a high level, without having to parse the source code of the actual package.

      3. Cardinality: Optional single instance

      4. Tag: “ShortDesc:”

      5. RDF: /RDF/SPDXDoc/Describes/Package/ShortDesc

      6. Data Format: free form text that can span multiple lines

      7. Example: ShortDesc: gnu c library

    10. Description

      1. Purpose: This field is a more detailed description of the package. It may be extracted from the packages itself.

      2. Intent: Here, the intent is to provide technical readers/reviewers with a detailed technical explanation of the functionality, anticipated use, and anticipated implementation of the package. This field may also include a description of improvements over prior version of the package, where applicable.

      3. Cardinality: Optional single instance

      4. Tag: “Description:”

      5. RDF: /RDF/SPDXDoc/Describes/Package/Description

      6. Data Format: free form text than can span multiple lines.

      7. Example: Description: …..
