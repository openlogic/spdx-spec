4. Index of Non Standard License(s) Detected

This section is used for any detected or declared licenses that are NOT one of the standard licenses. One instance should be created for every unique license detected in package that does not match one of the standard license short forms from Appendix I. Each license instance should have the following fields.

Fields:

    1. Identifier

      1. Purpose: Provide a unique identifier for the packages and files sections to refer to non standard license text detected in the package and reproduced here to aid analysis.

      2. Intent: Here, we seek to identify in whole or in part portions of the package which are licensed under unfamiliar and/or uncommon licenses.

      3. Cardinality: mandatory single instance

      4. Tag: "LicenseID"

      5. RDF: /RDF/SPDXDoc/Describes/License/LicenseID

      6. Data Format: "FullLicense-"N where N is a unique ascending numeric value.

      7. Example: LicenseID: FullLicense-1

    2. Extracted Text

      1. Purpose: Provide a copy of the actual text of the license extracted from the package that is associated with the License ID to aid in analysis.

      2. Intent: Here, the actual license text included in the package serves as confirmation that the license, and version, named in the header files of the package is correct.

      3. Cardinality: Mandatory single instance

      4. Tag: "LicenseText:"

      5. RDF: /RDF/SPDXDoc/Describes/License/LicenseText

      6. Data Format: free form text field that may span multiple lines.

      7. Example: LicenseText: .....
